First parts of a GUI available in "development" branch

Tue, Feb 12 2013, 12:11

Now that I have a bit more time for programming I yesterday started reimplementing some parts of my GUI draft in Swing. The first draft was only a local try, and never appeard in the sources. It used SWT as framework. I finally decided to use Swing as it is available on all Java supported platforms. Further I liked it that Swing does not need any additional libraries to be included in the final jar file. SWT would have cost approx. 6MB extra for each supported OS.

The implemented GUI parts are a starting point for the GUI, and will soon be filled with live ;-) For developers convenience I also included necessary Eclipse project files in the "development" branch of the repository. However, I will continue supporting the build.xml file in the repo to maintain usability for non-Eclipse hackers.

Cheers,
Nico

Bug: Weird behaviour if file changes its size

Sun, Jan 27 2013, 13:37

This bug is not all new, but I didn't really check the programs behaviour in this case until yesterday.

The problem:

Let's say you have a file in your shared directory which is 50MB in size. The default chunk size Peergroup uses is 512kB. If the file changes and and afterwards its size differs more than 512kB from it's original size, Peergroup will go weird.

So basically the weird behavior is caused if a file has more or less chunks after an update. In this case Peergroup downloads wrong chunks from the peer with the updated file, then compares the hashes of the wrong chunks with the hashes which would be correct, and finally attempts to redownload these (wrong) chunks because of the failed hash check.

I will work hard to fix this soon, but currently my exam preparations have a higher priority.

To spread some hope and happiness, the next update will not just fix this bug, it will also introduce these features:

  • Server mode (keep data available permanently if no other computers are online)
  • Switch between "No Encryption", "AES-128", "AES-256"
  • Improved detection of local IP (now also works on Linux computers)

    Released Version 0.05

    Together with the new website I release the new version 0.05 of PG. There are quite some features you might have been waiting for:

    Change Log:

    • Encrypted data transfers (using AES-128) (AES-256 will come in 0.06)
    • Settins are now stored in config.xml (no need for long start script anymore)
